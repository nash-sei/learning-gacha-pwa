# ガクモン 大人用問題 作成指示書（3AI共通）

> 目的：大人向けの問題集（一般教養・雑学・常識）を、ChatGPT・Grok・Claude の3つのAIに分担して作らせ、
> 学習アプリ「ガクモン」に取り込めるよう、形式を完全にそろえるための指示書。
> 作成日：2026-06-18

---

## 0. 全体像（最初に読む）

- **合計200問**。3つのジャンルに分けて、3つのAIが担当する（重複防止）。

| AI | ジャンル | 問題数 | IDの頭文字 | 出力ファイル名 |
|---|---|---|---|---|
| ChatGPT | 一般教養 | 67問 | `kyoyo-001`〜`kyoyo-067` | `adult-kyoyo.json` |
| Grok | 雑学 | 67問 | `zatsugaku-001`〜`zatsugaku-067` | `adult-zatsugaku.json` |
| Claude | 常識 | 66問 | `joshiki-001`〜`joshiki-066` | `adult-joshiki.json` |

- 各AIは **自分のジャンルだけ** を作る。他ジャンルには手を出さない。
- 出力は **JSONファイル1つ**（後述の形をそのまま守る）。
- 答え方は **基本4択**、ときどき **◯✕（2択）** を混ぜてよい。
- 対象は **日本の一般的な大人**。学校の教科書の延長ではなく、教養・雑学・常識から出題する。

> ⚠️ アプリ取り込みの前提：このJSONはそのままアプリに自動で入るわけではない。
> 受け取り後、テックチームが「変換スクリプト＋大人モード」をアプリ側に用意して取り込む。
> この指示書の役割は「3AIが作る問題の形を1つにそろえること」。

## 1. 出力ファイルの形（JSON）

ファイル全体は次の形。`meta`（このファイルの情報）＋ `questions`（問題の配列）。

```json
{
  "meta": {
    "title": "ガクモン 大人用問題：一般教養",
    "author": "ChatGPT",
    "category": "一般教養",
    "version": "1.0",
    "count": 67
  },
  "questions": [
    {
      "id": "kyoyo-001",
      "category": "一般教養",
      "difficulty": "normal",
      "question": "日本一高い山・富士山の標高は約何メートル？",
      "choices": ["3776m", "3015m", "2999m", "4017m"],
      "answerIndex": 0,
      "explanation": "富士山の標高は3776m。日本一高い山で、世界文化遺産にも登録されている。"
    }
  ]
}
```

## 2. 問題1問の項目（スキーマ）

| 項目（キー） | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | 文字列 | ✅ | 一意のID。担当ごとの頭文字＋3桁ゼロ埋め連番（例 `kyoyo-001`）。重複・欠番なし。 |
| `category` | 文字列 | ✅ | `"一般教養"` / `"雑学"` / `"常識"` のいずれか（自分の担当で固定）。 |
| `difficulty` | 文字列 | ✅ | `"easy"` / `"normal"` / `"hard"` のいずれか。 |
| `question` | 文字列 | ✅ | 問題文。1文〜2文で簡潔に。 |
| `choices` | 文字列の配列 | ✅ | 選択肢。**4択なら4個**、**◯✕なら必ず `["正しい", "誤り"]` の2個**。 |
| `answerIndex` | 整数 | ✅ | 正解の位置。**0から数える**（先頭が正解なら `0`、2番目なら `1`）。 |
| `explanation` | 文字列 | ✅ | 解説。1〜3文。答えだけでなく「なぜ」や一言の豆知識を添える。 |

---

## 3. 答え方のルール（4択＋◯✕）

- **基本は4択**。選択肢を必ず4つ用意する。
- **◯✕（2択）も混ぜてよい**。その場合 `choices` は必ず `["正しい", "誤り"]` の2つだけにする。
- **3択は使わない**（4択 か ◯✕ の2種類だけ）。
- ◯✕は気分転換用。**自分の担当の中で多くても15問まで**、残りは4択にする。
- どの問題も **正解はちょうど1つ**。`answerIndex` はその正解の位置（0始まり）。

---

## 4. ID・難易度・問題数のルール

- **ID**：自分の頭文字＋3桁ゼロ埋め連番。飛ばさず `001` から順番に。
  - ChatGPT：`kyoyo-001` 〜 `kyoyo-067`
  - Grok：`zatsugaku-001` 〜 `zatsugaku-067`
  - Claude：`joshiki-001` 〜 `joshiki-066`
- **難易度の目安（67問なら）**：`easy` 約17問 / `normal` 約33問 / `hard` 約17問。
  - `easy`＝多くの大人が知っている
  - `normal`＝教養として標準。知っている人は知っている
  - `hard`＝少し詳しい人向け。意外性のある問題
- **問題数はぴったり**：67問担当は67問、66問担当は66問。多すぎ・少なすぎ禁止。`meta.count` と実際の数を一致させる。

## 6. 内容の品質ルール（最重要）

- **事実は正確に。** 間違った答えを正解にしない。あいまい・諸説ある題材は避ける。
- **年で変わる事実は避ける**（例：現職の役職者、最新の記録、細かい人口）。どうしても使うなら問題文に「2020年時点で」など基準を書く。
- **正解は1つだけ。** 「すべて正しい」「上記のどれでもない」は使わない。
- **選択肢の作り方**：紛らわしすぎず、当てずっぽうでは当たらない程度。明らかに冗談の選択肢ばかりにしない。
- **不適切な題材は避ける**：政治的に偏った話・宗教・差別・性的・残酷な内容など。楽しく学べる範囲で。
- **同じファイル内で内容が重複しない**こと（似た問題の連発・同じ答えの繰り返しを避ける）。
- **解説は教育的に**：答えを言うだけでなく、理由や関連する一言の知識を添える。1〜3文。

---

## 7. JSON記法の注意（ここを守らないと取り込めない）

- **文字コードは UTF-8**。
- **正しいJSON**であること：
  - コメント（`//` や `/* */`）は **書かない**。
  - **末尾カンマ禁止**（配列・オブジェクトの最後の要素のうしろにカンマを付けない）。
  - キーは必ず **ダブルクォート** `"..."` で囲む。
  - 文字列内のダブルクォートは `\"`、改行は `\n` でエスケープ。
- **出力はJSONだけ**。前後に説明文・あいさつを付けない（コードブロックで囲むのは可）。

## 9. 提出前チェックリスト（AI自身が確認）

- [ ] 問題数は担当どおりか（67 or 66）。`meta.count` と一致しているか。
- [ ] IDは頭文字＋3桁連番で、重複・欠番がないか。
- [ ] `choices` は4個、または `["正しい","誤り"]` の2個になっているか。
- [ ] `answerIndex` は 0以上で、選択肢の数より小さいか（正しい位置を指しているか）。
- [ ] `difficulty` は easy / normal / hard のどれかか。
- [ ] 事実は正確か。年で変わる情報を入れていないか。
- [ ] 自分の担当ジャンルから外れていないか。
- [ ] JSONとして正しいか（末尾カンマなし・コメントなし・ダブルクォート）。
- [ ] 出力はJSONだけか（余計な前置きを付けていないか）。
